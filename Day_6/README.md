# Day 6
Pada pertemuan ke-6 kita akan belajar **Facade**
- Facade --> Stuctural Pettern  
    Contoh kasus analogi dari _Facade_ adalah ketika seorang pengunjung datang ke sebuah restoran dan ingin memesan makanan, maka akan ada 2 object yang bekerja waiter dan juga chef. Jika tanpa facade pelanggan akan memanggil 2 object tersebut untuk mencatat/mengantarkan pesanan dan memasak apa yang dipesan, tentu saja itu bukan hal yang tidak kita inginkan. Maka kita perlu alur yang lebih tersusun yaitu menggunakan _Facade_ sebuah class untuk menjadi jembatan antara pelanggan dengan object karyawan.  
    Fungsinya memberikan alur yang lebih tersusun.

    > Kesimpulan : bila ingin pelanggan(Main.java) tidak langsung mengakses object karyawan(chef dan waiter)  
    Contoh kode : [Facade](src/facade/)
